# Contributing

Thanks for helping improve API Test Kit.

## How to contribute

- Open an issue first for larger changes.
- Keep pull requests focused.
- Add or update tests for behavior changes.
- Update documentation when command usage changes.
- Run `pytest` before submitting a pull request.

## Code style

- Keep Python code simple and readable.
- Follow the existing naming and formatting style.
- Prefer small functions with clear inputs and outputs.

## Reporting bugs

Please include:

- The command you ran.
- The expected behavior.
- The actual behavior.
- A minimal sample project or traceback when possible.

## Security

Do not include secrets, tokens, or private credentials in issues or pull requests.
